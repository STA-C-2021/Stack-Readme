## ⭐️ Commit Message Convention
### 1. **각 파트별 package에 commit!**
### 2. **[커밋 타입]은 아래에 맞추어 작성하기**
>[ADD] : 코드나 테스트를 추가했을 때  
>[FIX] : 버그를 수정했을 때  
>[REMOVE] : 코드를 제거했을 때  
>[UPDATE] : 코드를 수정했을떄  
>[DOCS] : 문서를 수정했을 때  
>[RENAME] : 이름을 변경했을 때  
>[MOVE] : 코드를 이동헀을 때  
ex)[FIX]Dont access db
### 3. **본문은 다음과 같이 작성**
>- 왜 커밋했는지 설명.
>- 버그 수정의 경우 원래 어떤 문제가 있었는지 설명.
